# web/ — TypeScript Browser UI

**Phase 3 target.** This directory will contain the TypeScript browser application.

## Status

Phase 0 skeleton only. No TypeScript code yet. The current browser tool is `index.html`
at the repo root (vanilla JS, all-in-one file).

## Planned structure (Phase 3)

```
web/
  package.json        # npm deps: typescript, vite, @types/...
  tsconfig.json       # strict mode, ESNext, DOM lib
  vite.config.ts      # static build, WASM plugin
  src/
    index.html        # shell HTML (loads the TS entry point)
    main.ts           # entry: initDOM + setMode + event wiring
    types.ts          # SwatchOptions discriminated union, PresetObject, etc.
    ui/
      settings.ts     # all sidebar controls (shape/bg/format/etc.)
      render.ts       # results grid, cards, dashboard
      presets.ts      # preset modal, applyPreset(), savePreset()
      drag-drop.ts    # drag-over/drop handlers
    intake/
      files.ts        # file + folder + ZIP intake (JSZip, zero-install)
      excel.ts        # SheetJS intake, sheet/column detection
      urls.ts         # URL bulk input
    worker/
      pool.ts         # WorkerPool: n workers, task queue
      swatch.worker.ts # OffscreenCanvas worker calling WASM core
    pipeline/
      swatch.ts       # orchestrate pool calls, manage st.items
      transform.ts    # Image Transformer mode
    pkg/              # WASM build output (generated by wasm-pack, gitignored)
```

## Build (Phase 3+)

```bash
cd web
npm install
npm run build        # -> dist/ (static, openable with no server)
npm run typecheck    # tsc --noEmit
npm run dev          # Vite dev server (for development only)
```

## Zero-install constraint

The `dist/` output must open directly in a browser with `file://` — no server required.
This means:
- No dynamic imports of remote scripts at runtime.
- WASM loaded via `fetch` with a relative path (works on `file://` in Chrome/Edge/Firefox).
- JSZip and SheetJS bundled into the output (not loaded from CDN).
