# fixtures/

Test images and sample CSVs for the parity test suite.

## Required fixtures (Phase 1 will add these)

| File | Description | Why needed |
|------|-------------|------------|
| `on_white.jpg` | Product photo on plain white background | Tests smart crop + white bg detection |
| `transparent.png` | Image with alpha transparency | Tests alpha-aware bbox, transparent bg output |
| `landscape.jpg` | Wide landscape (>2:1 ratio) | Tests non-square ratios |
| `portrait_exif.jpg` | Portrait photo WITH EXIF orientation tag (rotated 90°) | Tests EXIF auto-rotate (bug #5) |
| `large_4000px.jpg` | ~4000×4000 JPEG | Tests Image.draft() fast decode, large-image pipeline |
| `urls_with_header.csv` | 3 rows + header row, URL column auto-detectable | Tests CSV with header |
| `urls_no_header.csv` | 3 rows, first row is a URL (no header) | Tests headerless CSV detection |

## Fixture sourcing

All test images are either:
- Public domain / CC0 (Unsplash, Pixabay, Wikimedia Commons), or
- Synthetically generated by the Phase 1 test setup script.

The EXIF-rotated fixture must have actual EXIF Orientation ≠ 1 (e.g., 6 = 90° CW).
Use `exiftool` or PIL to verify:
```
python -c "from PIL import Image; i=Image.open('portrait_exif.jpg'); print(i._getexif().get(274))"
```
Expected output: `6` (90° CW) or `8` (90° CCW).

## Adding new fixtures

Fixtures must be deterministic (same bytes on every run). Do NOT add fixture images
that are fetched at test time — embed them in the repo so tests work offline.
Max size per fixture: 2 MB. Total fixtures directory: < 20 MB.
